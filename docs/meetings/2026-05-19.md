## API Work Needed

Start with an endpoint or client call that can return nearby utterances for a selected utterance.

The useful response shape should include:

- the selected utterance;
- a configurable number of utterances before it;
- a configurable number of utterances after it;
- meeting metadata;
- city metadata;
- speaker or speaker segment metadata, when available.

A practical first version could return 15 utterances before and 15 after the selected utterance.

## UI Notes

- The UI should support seeded random review order.
- A reviewer should be able to enter a seed and get the same sequence every time.
- The frontend can request review items in batches instead of loading everything at once.
- The frontend should preload a small window around the current item, especially the next few examples.
- Include, exclude, and uncertain decisions should be exportable.
- Error category labels should be exportable.
- Aggregate stats can be computed from the exported review data before adding heavier storage.

## Dataset Notes

- The new CSV export includes the fields that were previously missing.
- LLM/task edits should stay in the candidate pool.
- Human edits and LLM/task edits can later be compared or weighted differently.
- Rows that did not need correction may also be useful as clean examples.
- The dataset selection step should happen after the taxonomy and review workflow have produced enough signal.
