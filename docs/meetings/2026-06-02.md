## Discord follow-up — train/test split

- **Overfitting risk:** fine-tuning currently pulls randomly from the whole
  correction pool, so a test on some council may contain utterances that were in
  training — a good result could just be memorisation.
- **Consensus (Christos, agreed):** split **whole meetings**, not individual
  utterances — every utterance of a meeting belongs entirely to train, val, or
  test. → [decisions/data.md](../decisions/data.md#2026-06---split-trainvaltest-by-whole-meeting-not-by-utterance).
- **Baseline first (Christos):** measure two things before any finetuning — how
  Gladia does, and how Whisper does *without* finetuning.
- **Open: fixed benchmark vs random set (Christos vs Ethniki).** A fixed set of
  meetings gives stable, comparable metrics across runs; a random set varies
  (WER 20 → 23 → 16) and needs statistics to average out. → open question in
  [decisions/data.md](../decisions/data.md#open).
